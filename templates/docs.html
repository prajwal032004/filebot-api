API Documentation

Complete guide to integrating Image API into your applications. Store, organize, and retrieve images and PDFs with ease.

Introduction

Welcome to the Image API documentation. This RESTful API allows you to store images and PDFs, organize them in folders, and retrieve them programmatically from any application.

Fast & Reliable

Built on Flask with PythonAnywhere hosting for 99.9% uptime and low-latency responses.

Secure Authentication

Every request is authenticated with your unique API key using the X-API-Key header.

JSON Responses

All endpoints return clean, structured JSON with proper HTTP status codes.

No Limits

Unlimited folders and files. Each file can be up to 10MB in size.

Getting Started

Follow these steps to start using the Image API in your application.

Quick Start Guide
  1. Create an account at imageapi.pythonanywhere.com
  2. Get your API key from the Settings page
  3. Include the key in the X-API-Key header
  4. Start making requests to the API endpoints

Authentication

All API requests must include your API key in the request header for authentication.

Keep Your API Key Secure
Never share your API key publicly or commit it to version control. Use environment variables to store it securely in your applications.

Header Format

Authentication Header 
X-API-Key: {% if current_user.is_authenticated %}{{ current_user.api_key }}{% else %}your_api_key_here{% endif %}

Example with Different Languages

Python (requests) 
import requests

API_KEY = "{% if current_user.is_authenticated %}{{ current_user.api_key }}{% else %}your_api_key_here{% endif %}"
BASE_URL = "https://imageapi.pythonanywhere.com"

# Make authenticated request
headers = {"X-API-Key": API_KEY}
response = requests.get(f"{BASE_URL}/api/folders", headers=headers)

if response.status_code == 200:
    data = response.json()
    print(f"Found {len(data['data'])} folders")
else:
    print(f"Error: {response.status_code}")
cURL 
# Set your API key
API_KEY="{% if current_user.is_authenticated %}{{ current_user.api_key }}{% else %}your_api_key_here{% endif %}"

# Make authenticated request
curl -X GET https://imageapi.pythonanywhere.com/api/folders \
  -H "X-API-Key: $API_KEY" \
  | jq '.'
PHP 


$apiKey = "{% if current_user.is_authenticated %}{{ current_user.api_key }}{% else %}your_api_key_here{% endif %}";
$baseUrl = "https://imageapi.pythonanywhere.com";

// Initialize cURL
$ch = curl_init("$baseUrl/api/folders");

// Set headers

    "X-API-Key: $apiKey"

// Execute request
$response = curl_exec($ch);
$data = json_decode($response, true);

echo "Found " . count($data['data']) . " folders";

curl_close($ch);
?>

Base URL

All API requests should be made to the following base URL:

Production URL 
https://imageapi.pythonanywhere.com
HTTPS Only
All API requests must be made over HTTPS. HTTP requests will be rejected for security reasons.

Rate Limits

To ensure fair usage and maintain service quality, API requests are rate-limited per account.

Time Period Request Limit Details
Per Hour 50 requests Resets every hour
Per Day 200 requests Resets at midnight UTC
Registration 5 per hour New account creation limit
Login 10 per hour Authentication attempts limit
Rate Limit Headers
Each API response includes rate limit information in the headers:
  • X-RateLimit-Limit: Maximum requests allowed
  • X-RateLimit-Remaining: Requests remaining in current window
  • X-RateLimit-Reset: Time when the limit resets (Unix timestamp)

Error Handling

The API uses standard HTTP status codes and returns error messages in JSON format.

Status Code Meaning Description
200 OK Request succeeded
201 Created Resource created successfully
400 Bad Request Invalid request parameters
401 Unauthorized Missing or invalid API key
403 Forbidden Access denied to resource
404 Not Found Resource does not exist
429 Too Many Requests Rate limit exceeded
500 Internal Server Error Server-side error occurred

Error Response Format

Error Response Example 
{
  "status": "error",
  "message": "Invalid API key",
  "code": 401
}

Best Practices

Follow these guidelines to ensure optimal performance and security when using the API.

Secure Your API Key

Store your API key in environment variables. Never hard-code it in your source code or commit it to version control systems like Git.

Handle Errors Gracefully

Always check the HTTP status code and handle errors appropriately. Implement retry logic with exponential backoff for 5xx errors.

Respect Rate Limits

Monitor rate limit headers in responses and implement throttling in your application to avoid hitting limits.

Optimize File Sizes

Compress images before uploading to reduce storage and bandwidth usage. Consider using appropriate formats (WebP for images, optimized PDFs).
